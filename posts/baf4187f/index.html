加班一周记

2021-01-27
字数统计: 1.8k   |   阅读时长≈ 6 分钟

上周周二开会说要临时上个项目，只有一周时间要上线，于是苦逼的一星期就从那天开始了！

当天下午所有人停下手中事情，全部投入到新的项目中去，分到我手里的活虽然不是很难，但却是我从来没有接触过的，公众号 H5 调用微信扫一扫,然后显示一些接口返回的数据信息。

Day1: 2021-01-19 星期二

任务: 实现调用扫一扫 Demo。

当天下午开始我就在考虑用什么框架来实现这个简单的功能。

说实话手机端的 H5 以前使用的 Ionic 都已经是四五年前的事情了，只知道已经更新了几个大版本肯定不能再用了。打算用 Bootstrap 来实现一个简单的 Demo 但目前 Bootstrap 只在 SpringBoot 的 Web 页面上使用过，手机端能力不是很清楚，但也基于现有项目写了一个简单的 Html 页面调用 JS-SDK，结果发现需要一些初始化参数签名之类的坑要平。

由于没使用过微信 JS-SDK，也不熟悉他的使用流程，再加之会议上说的要获取使用用户的 OpenId。于是看了看微信的文档一下午就过去了。

加班到晚上 9 点多下班回家。

Day2: 2021-01-20 星期三

任务: 实现前端基本原型页面。

了解到 OpenId 和 JS-SDK 完全是两个相互独立的业务和流程后，重点就放在了构建前端原型界面上。

每个人都有自己的分工。我作为一个标准的前端，自认为前端页面是一个很容易的事情，于是把时间主要花费在了跑通整个调用扫一扫流程上面。

上午在框架选型上就纠结了半天，不知道到底用 Vue 还是 React，Vue 是近期做后台管理界面用起来相对简单顺手的框架，但做手机端 H5 页面我心里也没谱。于是想用 React 来实现，使用官方脚手架工具 Create-React-App 创建了基本的项目框架，但发现找不到平时很熟悉的 UI 控件。后来又用京东的 Taro 创建了项目，结果在使用 Vue 还是 React，使用 Ts 还是 Js，使用 Less 还是 Scss 上来来回回尝试了好几次。

想要配置 JS-SDK，又没人有空给我提供接口。我只能拿我自己的 Node 项目迅速的添加了一个模块把微信的 Api 调用流程的坑趟平了。有了配置和签名但也不是一帆风顺的把扫一扫调用成功的，由于粗心把微信后台参数配置错误导致我一直失败。后来催我要我的原型界面结果到快下班我才决定暂时放弃尝试，先把界面搭建出来。

又一次加班到 9 点多下班回家，回家接着把未完成的工作做完都已经凌晨一点多了。当然主要是因为首次使用 Taro 这个框架，涉及到一些基本的页面跳转、页面传参、接口请求、UI 使用花费了不少时间。

Day3: 2021-01-21 星期四

任务: 帮助后台对接接口。

完成了基本的原型界面，又说要优化一下。没有设计图，我也不知道哪里需要改进，昨天熟悉了 UI 框架后感觉也没什么难度，只要给个图还是很容易实现出来的。

之前储备了一周的 Vue 后台管理端这次算是派上了用场。外边派来的人复制粘贴完界面却没有后端项目的权限，基本上完成的后端接口我在我本地调用起来还是比较容易。由于熟悉这边的项目框架所以一下午都在帮忙给后台管理端对接口上面，完成了 5 个模块的接口增删改查对接。

加班到 9 点多下班回家。

Day4: 2021-01-22 星期五

任务: 修复 Bug

不同的人有不同的思维方式，不同的编码习惯，也会产生不同的 Bug。更何况还有一些需求明确性和功能优化性改动。

虽然项目不是很复杂，功能也都只是一些增删改查的基础功能，而且每个模块大部分都一样。但光是一些数据长度校验和选填必填改动貌似也花费了不少的时间。

又一次加班多9点多下班回家。

Day5: 2021-01-23 星期六

任务: 上线整体项目包括前端扫码。

昨天的测试基本扫清了绝大部分问题，所以今天做最后的测试然后上线运行。

我知道接下来主要的工作就是前端功能，就提前催了几次高保真设计图。下午后台终于闲下来和我对接微信扫一扫这一块流程，不知道他们从哪个项目拷贝出来一段代码然后告诉我配置好参数就能获取到签名配置。看他对这个业务也并不是很熟悉，我就将我了解到的东西按照流程重新给他讲解了一遍，突然才发现有项配置与教程上不符，改过之后就能正常运行整个扫码流程了。

心里这块石头总算落地了，生怕这个流程没搞定影响了整个项目进度。

下午快下班时前端设计终于做好了，说是要一起上线那就只有加班赶出来。界面并没有花费很长时间，主要是与设计和后台对数据浪费了不少时间。要么没有测试数据，要么测试数据不完整不全面，找人加个数据推来推去的，几番磨合才把每个数据核对上。不过时间已经很晚了。

十二点左右部署了前端版本，大家都兴奋的测来测去。不过也测出来不少问题，数据查询不成功，接口报错，界面显示错位。又来来回回改了几次终于在一点多有了稳定的版本上线了。

凌晨一点半。下班回家。

Day6: 2021-01-24 星期天

项目成功上线了，但这一周的忙碌感觉身体很疲惫。除了吃饭就在家整整睡了一整天。

后记

这次项目在框架选型和流程调试上浪费了不少时间，也因为粗心大意没仔细看教程走了不少弯路，但总体上没有拉下进度。

我也从一个手机前端完完全全转变成了Web前端，迅速的承接大部分的前端改动，但想想上个月的工资单真的不知道这样的转变是值还是不值。
